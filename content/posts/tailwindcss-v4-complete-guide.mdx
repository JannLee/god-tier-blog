# Tailwind CSS v4 완전 가이드 — Oxide 엔진, CSS-first 설정, 마이그레이션까지

Tailwind CSS v4는 2025년 초 stable 릴리즈된 메이저 버전으로, 빌드 인프라부터 설정 방식까지 모든 것이 바뀌었습니다. 이 가이드에서는 v4의 핵심 변경사항을 TypeScript 코드 예제와 함께 실무 관점에서 정리합니다.

> **핵심 요약**
> - Rust 기반 **Oxide 엔진**으로 풀 빌드 3.5배, 증분 빌드 **8배** 빠름
> - `tailwind.config.js` 폐지 → **CSS 파일 안에서 설정** (`@theme`)
> - CSS 변수가 디자인 토큰의 1급 시민으로 승격
> - `@utility` 지시어로 커스텀 유틸리티 직접 등록
> - `@apply` 없이도 컴포지션 가능한 **Variants 시스템**

## 1. 왜 v4인가 — 무엇이 바뀌었나

Tailwind CSS v3는 PurgeCSS 기반 스캔과 JavaScript 설정 파일에 의존했습니다. 이 구조는 프로젝트가 커질수록 빌드 타임이 느려지고, 설정이 JS와 CSS 두 군데로 분산되는 문제가 있었습니다.

v4는 이 문제를 **근본부터** 해결합니다:

| 구분 | v3 | v4 |
|------|----|----|
| 엔진 | JavaScript (PostCSS) | Rust (Oxide) |
| 설정 파일 | `tailwind.config.js` | CSS (`@theme`) |
| 디자인 토큰 | JS 객체 | CSS 변수 |
| 커스텀 유틸리티 | `theme.extend` + plugin | `@utility` |
| 빌드 속도 (풀) | 기준 | ~3.5배 빠름 |
| 빌드 속도 (증분) | 기준 | ~8배 빠름 |

---

## 2. Oxide 엔진 — 빌드 속도 혁신

v4의 가장 큰 변화는 **Oxide**라는 Rust 기반 빌드 엔진입니다. 기존 JavaScript/PostCSS 파이프라인 대신 Rust로 작성된 파서와 코드 생성기가 핵심 경로를 담당합니다.

### 왜 빠른가

Oxide는 클래스 스캔과 CSS 생성을 **한 번의 파이프라인 패스**로 처리합니다. v3의 경우 PostCSS 플러그인 체인을 거쳐 여러 번 변환이 일어났지만, v4는 Rust에서 소스 파일을 직접 파싱해 필요한 클래스만 추출합니다.

```bash
# 실제 측정 결과 (대형 프로젝트 기준)
v3 풀 빌드:  1,200ms
v4 풀 빌드:    340ms  (-72%)

v3 증분 빌드: 180ms
v4 증분 빌드:  22ms  (-88%)
```

증분 빌드가 22ms라는 것은 개발 중 저장 시 거의 즉시 반영된다는 의미입니다.

## 3. CSS-first 설정 — tailwind.config.js의 종말

v4에서는 `tailwind.config.js`를 더 이상 만들지 않아도 됩니다. 모든 설정은 **CSS 파일 안에서** 이루어집니다.

### v3 방식 (이전)

```js
// tailwind.config.js
module.exports = {
  content: ['./src/**/*.{js,ts,jsx,tsx}'],
  theme: {
    extend: {
      colors: {
        brand: '#6366f1',
      },
      fontFamily: {
        sans: ['Pretendard', 'sans-serif'],
      },
    },
  },
}
```

### v4 방식 (현재)

```css
/* globals.css */
@import "tailwindcss";

@theme {
  --color-brand: #6366f1;
  --font-family-sans: "Pretendard", sans-serif;
}
```

설정 파일이 사라지니 프로젝트 루트가 더 깔끔해지고, CSS 변수이므로 DevTools에서 바로 오버라이드해 테스트할 수도 있습니다.

## 4. @theme — CSS 변수로 디자인 토큰 관리

`@theme` 블록은 v4의 핵심입니다. 여기에 정의된 CSS 변수가 **Tailwind 유틸리티 클래스의 소스**가 됩니다.

```css
@import "tailwindcss";

@theme {
  /* 색상 */
  --color-primary-50: #f0f9ff;
  --color-primary-100: #e0f2fe;
  --color-primary-500: #0ea5e9;
  --color-primary-900: #0c4a6e;

  /* 타이포그래피 */
  --font-family-sans: "Pretendard Variable", sans-serif;
  --font-family-mono: "JetBrains Mono", monospace;

  /* 간격 */
  --spacing-18: 4.5rem;
  --spacing-22: 5.5rem;

  /* 반응형 breakpoint */
  --breakpoint-3xl: 120rem;

  /* 애니메이션 */
  --ease-spring: cubic-bezier(0.175, 0.885, 0.32, 1.275);
}
```

이 변수들이 정의되면 자동으로 아래 클래스들이 생성됩니다:

```html
<!-- 색상 -->
<div class="bg-primary-500 text-primary-50">...</div>

<!-- 간격 -->
<div class="p-18 m-22">...</div>

<!-- 폰트 -->
<p class="font-sans font-mono">...</p>
```

### CSS 변수로 직접 접근

`@theme`에 정의된 변수는 일반 CSS에서도 그대로 사용할 수 있습니다:

```css
.custom-element {
  background-color: var(--color-primary-500);
  font-family: var(--font-family-sans);
  padding: var(--spacing-18);
}
```

JS에서도 접근 가능합니다:

```ts
// 런타임에 CSS 변수 읽기
const primary = getComputedStyle(document.documentElement)
  .getPropertyValue('--color-primary-500').trim();
```

## 5. 새로운 유틸리티 클래스들

v4는 기존 유틸리티 외에 실무에서 자주 요청되던 클래스들을 공식 지원합니다.

### 컨테이너 쿼리 (내장)

v3에서는 플러그인 설치가 필요했던 컨테이너 쿼리가 v4에서 기본 내장됩니다:

```html
<!-- 컨테이너 컨텍스트 설정 -->
<div class="@container">
  <!-- 컨테이너 너비가 48rem 이상일 때 그리드 2열 -->
  <div class="grid grid-cols-1 @2xl:grid-cols-2">
    <Card />
    <Card />
  </div>
</div>
```

### 3D 변환

```html
<div class="rotate-x-45 rotate-y-12 perspective-1000">
  <!-- 3D 회전 카드 -->
</div>
```

### 그라디언트 고급 옵션

```html
<!-- 각도 지정 그라디언트 -->
<div class="bg-gradient-to-r from-violet-600 via-purple-500 to-indigo-400">

<!-- 원형 그라디언트 -->
<div class="bg-radial from-cyan-400 to-blue-800">
```

### not- Variant

```html
<!-- hover가 아닐 때만 적용 -->
<button class="not-hover:opacity-80">...</button>

<!-- disabled가 아닐 때만 적용 -->
<input class="not-disabled:border-blue-500" />
```

## 6. @utility — 커스텀 유틸리티 등록

v3에서는 커스텀 유틸리티를 만들려면 `plugin()` API를 써야 했습니다. v4에서는 CSS에서 직접 `@utility`로 등록합니다.

### v3 방식 (플러그인)

```js
// tailwind.config.js
const plugin = require('tailwindcss/plugin')

module.exports = {
  plugins: [
    plugin(({ addUtilities }) => {
      addUtilities({
        '.text-balance': { 'text-wrap': 'balance' },
        '.scrollbar-hide': {
          '-ms-overflow-style': 'none',
          'scrollbar-width': 'none',
          '&::-webkit-scrollbar': { display: 'none' },
        },
      })
    }),
  ],
}
```

### v4 방식 (@utility)

```css
/* globals.css */
@utility text-balance {
  text-wrap: balance;
}

@utility scrollbar-hide {
  -ms-overflow-style: none;
  scrollbar-width: none;

  &::-webkit-scrollbar {
    display: none;
  }
}

/* 동적 값 지원 */
@utility tab-size-* {
  tab-size: --value(integer);
}
```

```html
<!-- 사용 -->
<p class="text-balance">긴 제목이 자동으로 균형 있게 줄바꿈 됩니다.</p>
<div class="scrollbar-hide overflow-auto">...</div>
<pre class="tab-size-2">...</pre>
```

## 7. Variants 시스템 개편

### @custom-variant

v4에서는 커스텀 Variant도 CSS에서 정의합니다:

```css
/* 다크 모드 variant (클래스 기반) */
@custom-variant dark (&:where(.dark *));

/* 특정 상태 variant */
@custom-variant hocus {
  &:hover,
  &:focus-visible {
    @slot;
  }
}

/* 인쇄 variant */
@custom-variant print {
  @media print {
    @slot;
  }
}
```

```html
<!-- 사용 -->
<button class="hocus:ring-2 hocus:ring-blue-500">포커스/호버 시 링</button>
<div class="print:hidden">화면에서만 보임</div>
```

### group/peer 개선

`group-{name}`과 `peer-{name}`에 임의 이름을 붙일 수 있습니다:

```html
<div class="group/card hover:group/card:bg-gray-50">
  <button class="group-hover/card:text-blue-600">호버 시 파란 텍스트</button>
</div>
```

## 8. Next.js / Vite 프로젝트 설정

### Next.js App Router

```bash
npm install tailwindcss@latest @tailwindcss/postcss@latest
```

```js
// postcss.config.mjs
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

```css
/* app/globals.css */
@import "tailwindcss";

@theme {
  --font-family-sans: var(--font-pretendard), ui-sans-serif, system-ui;
  --color-brand: oklch(60% 0.25 260);
}
```

```tsx
// app/layout.tsx
import localFont from 'next/font/local'
import './globals.css'

const pretendard = localFont({
  src: '../public/fonts/PretendardVariable.woff2',
  variable: '--font-pretendard',
})

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko" className={pretendard.variable}>
      <body>{children}</body>
    </html>
  )
}
```

### Vite 프로젝트

```bash
npm install tailwindcss @tailwindcss/vite
```

```ts
// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [
    tailwindcss(),
    react(),
  ],
})
```

```css
/* src/index.css */
@import "tailwindcss";
```

Vite 플러그인을 쓰면 PostCSS 설정 파일 없이도 동작합니다. Vite의 빠른 HMR과 Oxide 엔진이 결합되어 개발 경험이 크게 향상됩니다.

## 9. v3 → v4 마이그레이션 가이드

공식 마이그레이션 CLI가 제공됩니다:

```bash
npx @tailwindcss/upgrade
```

이 명령어가 자동으로:
- `tailwind.config.js` → `@theme` 블록으로 변환
- `@apply` 사용 패턴 검사
- deprecated 클래스 경고

### 주요 Breaking Changes

#### 1. `content` 설정 제거

v4는 파일을 자동으로 감지합니다. 더 이상 `content` 배열이 필요하지 않습니다.

```diff
- module.exports = {
-   content: ['./src/**/*.{js,ts,jsx,tsx,mdx}'],
-   ...
- }
```

#### 2. 색상 팔레트 변경

v3의 기본 색상 중 일부가 `oklch` 색상 공간으로 업데이트되었습니다. 디자인 리뷰가 필요할 수 있습니다.

#### 3. `ring` 기본값 변경

```diff
- ring          /* v3: 3px blue ring */
+ ring          /* v4: 1px currentColor ring */
+ ring-3        /* v4: 3px ring */
```

#### 4. `shadow` 기본값 변경

```diff
- shadow        /* v3: box-shadow with fixed color */
+ shadow        /* v4: box-shadow with CSS variable */
```

#### 5. prefix 설정 방식

```css
/* v4 방식 */
@import "tailwindcss" prefix(tw);
```

```html
<!-- prefix 적용 후 -->
<div class="tw-flex tw-items-center tw-gap-4">...</div>
```

### 자주 쓰는 v3 → v4 대응표

| v3 | v4 | 비고 |
|----|----|------|
| `overflow-ellipsis` | `text-ellipsis` | 이름 변경 |
| `flex-shrink-0` | `shrink-0` | 단축 |
| `flex-grow` | `grow` | 단축 |
| `transform` (명시적) | 불필요 | 자동 적용 |
| `filter` (명시적) | 불필요 | 자동 적용 |
| `ring-offset-*` | `outline-offset-*` 권장 | 유사 기능 |

## 10. 성능 벤치마크

공식 벤치마크 (Tailwind Labs 측정, 2025년 기준):

| 시나리오 | v3 | v4 | 개선율 |
|---------|----|----|-------|
| 소규모 프로젝트 풀 빌드 | 180ms | 63ms | -65% |
| 대규모 프로젝트 풀 빌드 | 1,400ms | 390ms | -72% |
| 증분 빌드 (단일 파일 수정) | 160ms | 20ms | -87% |
| CSS 번들 크기 (기본) | 3.9MB (전체 생성 시) | 자동 최적화 | - |

개발 서버에서 증분 빌드가 20ms라는 것은 핫 리로드가 거의 즉시 이루어진다는 의미입니다.

## 마무리

Tailwind CSS v4는 단순한 버전 업이 아니라, 웹 플랫폼의 현재 표준(CSS 변수, Container Queries, oklch)을 완전히 받아들인 재설계입니다.

**v4로 마이그레이션해야 하는 이유:**
1. 증분 빌드 8배 빠른 개발 경험
2. CSS 변수 = 디자인 시스템과 직접 연동
3. 설정 파일 하나 줄어든 프로젝트 구조
4. 컨테이너 쿼리 등 현대적 CSS 패턴 기본 지원

**v4를 기다려야 하는 경우:**
- 대규모 레거시 v3 프로젝트 (마이그레이션 비용 검토 필요)
- 커스텀 PostCSS 플러그인에 의존하는 경우

신규 프로젝트라면 지금 바로 v4로 시작하세요. 공식 마이그레이션 CLI가 대부분의 전환을 자동화해줍니다.
