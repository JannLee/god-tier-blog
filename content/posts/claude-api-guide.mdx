# Claude API 사용법 완전 가이드 — 코드로 AI 통합하기

**Claude API(Anthropic API)**를 처음 써보려는 분들을 위한 단계별 실전 가이드입니다. API 키 발급부터 스트리밍 응답, 비용 최적화 전략까지 TypeScript와 Python 코드 예제를 곁들여 설명합니다.

> **요점 정리**
> - 환경 설정: `npm install @anthropic-ai/sdk` 또는 `pip install anthropic`
> - 첫 API 호출까지 10분 이내 완료 가능
> - Haiku 모델 기준 1,000토큰당 약 $0.001 — 실용적인 비용으로 프로덕션 운영 가능

## 1. Anthropic API 시작하기

### API 키 발급 방법

Claude API를 사용하려면 Anthropic Console에서 API 키를 발급받아야 합니다.

1. [console.anthropic.com](https://console.anthropic.com) 접속 후 회원가입 또는 로그인
2. 좌측 메뉴 **API Keys** → **Create Key** 클릭
3. 키 이름 입력 후 생성 — 키는 **한 번만 표시**되므로 즉시 안전한 곳에 저장

> ⚠️ API 키를 코드에 하드코딩하지 마세요. 환경 변수(`ANTHROPIC_API_KEY`)로 관리하는 것이 기본입니다.

### 개발 환경 설정

**TypeScript / Node.js:**

```bash
npm install @anthropic-ai/sdk
# 또는 pnpm add @anthropic-ai/sdk
```

`.env` 파일:

```bash
ANTHROPIC_API_KEY=sk-ant-api03-xxxxxx
```

**Python:**

```bash
pip install anthropic
# 또는 uv add anthropic
```

`.env` 파일:

```bash
ANTHROPIC_API_KEY=sk-ant-api03-xxxxxx
```

## 2. 메시지 API 기본 사용법

Claude API의 핵심은 `messages.create` 엔드포인트입니다. 사용자 메시지를 배열로 전달하면 모델이 응답을 생성합니다.

### TypeScript로 첫 요청 보내기

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

async function askClaude(question: string): Promise<string> {
  const message = await client.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 1024,
    messages: [
      {
        role: "user",
        content: question,
      },
    ],
  });

  // 응답에서 텍스트 추출
  const block = message.content[0];
  if (block.type !== "text") throw new Error("Unexpected response type");
  return block.text;
}

// 사용 예시
const answer = await askClaude("한국의 수도는 어디인가요?");
console.log(answer); // "한국의 수도는 서울입니다."
```

**응답 객체 주요 필드:**

| 필드 | 설명 |
|------|------|
| `id` | 요청 고유 ID (`msg_xxxxxxxx`) |
| `content` | 응답 블록 배열 (`text` 또는 `tool_use`) |
| `usage.input_tokens` | 입력 토큰 수 |
| `usage.output_tokens` | 출력 토큰 수 |
| `stop_reason` | 종료 이유 (`end_turn`, `max_tokens`, `tool_use`) |

### Python으로 첫 요청 보내기

```python
import anthropic
import os

client = anthropic.Anthropic(
    api_key=os.environ.get("ANTHROPIC_API_KEY")
)

def ask_claude(question: str) -> str:
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[
            {"role": "user", "content": question}
        ]
    )
    return message.content[0].text

# 사용 예시
answer = ask_claude("파이썬의 장점 세 가지를 알려주세요.")
print(answer)
```

## 3. 스트리밍 응답 구현

긴 텍스트를 생성할 때 스트리밍을 사용하면 **첫 토큰 도달 시간(TTFT)**을 크게 줄일 수 있습니다. 스트리밍 없이 2,000토큰짜리 응답을 기다리면 5~10초가 걸리지만, 스트리밍을 쓰면 0.3~0.5초 안에 첫 글자가 표시됩니다.

### TypeScript 스트리밍

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

async function streamResponse(prompt: string): Promise<void> {
  const stream = await client.messages.stream({
    model: "claude-sonnet-4-6",
    max_tokens: 2048,
    messages: [{ role: "user", content: prompt }],
  });

  // 청크 단위로 실시간 출력
  for await (const chunk of stream) {
    if (
      chunk.type === "content_block_delta" &&
      chunk.delta.type === "text_delta"
    ) {
      process.stdout.write(chunk.delta.text);
    }
  }

  // 최종 메시지 (사용량 확인용)
  const finalMessage = await stream.getFinalMessage();
  console.log("\n\n[토큰 사용량]", finalMessage.usage);
}

await streamResponse("REST API 설계 원칙을 자세히 설명해주세요.");
```

### Python 스트리밍

```python
import anthropic

client = anthropic.Anthropic()

def stream_response(prompt: str) -> None:
    with client.messages.stream(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        messages=[{"role": "user", "content": prompt}],
    ) as stream:
        for text in stream.text_stream:
            print(text, end="", flush=True)

    # 최종 메시지 정보
    final = stream.get_final_message()
    print(f"\n\n입력 토큰: {final.usage.input_tokens}")
    print(f"출력 토큰: {final.usage.output_tokens}")

stream_response("마이크로서비스 아키텍처의 장단점을 설명해주세요.")
```

## 4. 실전 활용 패턴 5가지

### 패턴 1: 시스템 프롬프트로 역할 부여

시스템 프롬프트는 Claude의 **행동 방식과 페르소나**를 정의합니다. 잘 쓰면 응답 품질이 극적으로 개선됩니다.

```typescript
const message = await client.messages.create({
  model: "claude-haiku-4-5-20251001",
  max_tokens: 512,
  system:
    "당신은 10년 경력의 시니어 백엔드 엔지니어입니다. " +
    "기술적으로 정확하고 간결하게 답변하며, 코드 예제를 적극 활용합니다. " +
    "불확실한 내용은 솔직히 모른다고 말합니다.",
  messages: [
    {
      role: "user",
      content: "PostgreSQL에서 N+1 문제를 해결하는 방법은?",
    },
  ],
});
```

### 패턴 2: 멀티턴 대화 구현

Claude API는 상태를 저장하지 않습니다. 대화 기록을 직접 관리해 넘겨야 합니다.

```typescript
type Message = { role: "user" | "assistant"; content: string };

async function chat(
  history: Message[],
  userMessage: string
): Promise<{ reply: string; history: Message[] }> {
  const newHistory: Message[] = [
    ...history,
    { role: "user", content: userMessage },
  ];

  const response = await client.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 1024,
    messages: newHistory,
  });

  const reply = (response.content[0] as { type: "text"; text: string }).text;

  return {
    reply,
    history: [...newHistory, { role: "assistant", content: reply }],
  };
}

// 사용 예시
let history: Message[] = [];
const r1 = await chat(history, "JavaScript의 클로저란 무엇인가요?");
history = r1.history;

const r2 = await chat(history, "방금 설명한 개념을 실제 코드로 보여주세요.");
console.log(r2.reply); // 이전 대화 맥락을 기억함
```

### 패턴 3: JSON 출력 강제하기

구조화된 데이터가 필요할 때 사용합니다. 프롬프트에 명시적으로 JSON 형식을 지정하면 신뢰성 있게 파싱 가능한 출력을 얻을 수 있습니다.

```python
import json

def extract_product_info(description: str) -> dict:
    response = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=512,
        messages=[{
            "role": "user",
            "content": f"""다음 상품 설명에서 정보를 추출하여 JSON으로 반환하세요.
반드시 아래 형식만 출력하고 다른 텍스트는 포함하지 마세요:

{{
  "name": "상품명",
  "price": 숫자(원),
  "category": "카테고리",
  "features": ["특징1", "특징2"]
}}

상품 설명: {description}"""
        }]
    )
    
    raw = response.content[0].text.strip()
    return json.loads(raw)

info = extract_product_info("삼성 갤럭시 S25, 256GB, 가격 120만원, 카메라 200MP, AI 기능 탑재")
print(info)
# {'name': '삼성 갤럭시 S25', 'price': 1200000, 'category': '스마트폰', 'features': [...]}
```

### 패턴 4: 도구(Tools) 호출 — Function Calling

Claude에게 함수를 선언하면, 필요할 때 해당 함수를 호출하도록 요청합니다.

```typescript
const tools: Anthropic.Tool[] = [
  {
    name: "get_weather",
    description: "특정 도시의 현재 날씨를 조회합니다.",
    input_schema: {
      type: "object",
      properties: {
        city: {
          type: "string",
          description: "도시 이름 (예: 서울, 부산)",
        },
      },
      required: ["city"],
    },
  },
];

const response = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  tools,
  messages: [{ role: "user", content: "서울 날씨 어때?" }],
});

// stop_reason === 'tool_use' 이면 도구 호출 필요
if (response.stop_reason === "tool_use") {
  const toolUse = response.content.find((b) => b.type === "tool_use");
  if (toolUse?.type === "tool_use") {
    console.log("호출할 함수:", toolUse.name); // "get_weather"
    console.log("인자:", toolUse.input); // { city: "서울" }
    // 실제 날씨 API 호출 후 결과를 messages에 추가해 재호출
  }
}
```

### 패턴 5: 이미지 처리 (Vision)

Claude는 이미지를 직접 받아 분석할 수 있습니다. Base64 인코딩 또는 URL을 지원합니다.

```python
import base64

def analyze_image(image_path: str, question: str) -> str:
    with open(image_path, "rb") as f:
        image_data = base64.standard_b64encode(f.read()).decode("utf-8")
    
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[{
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/png",
                        "data": image_data,
                    },
                },
                {
                    "type": "text",
                    "text": question
                }
            ]
        }]
    )
    return response.content[0].text

result = analyze_image("screenshot.png", "이 에러 메시지의 원인이 무엇인가요?")
print(result)
```

| 모델 | 입력 (MTok당) | 출력 (MTok당) | 적합한 용도 |
|------|-------------|-------------|-----------|
| **Claude Haiku 4.5** | ~$0.80 | ~$4.00 | 분류, 간단한 Q&A, 대량 처리 |
| **Claude Sonnet 4.6** | ~$3.00 | ~$15.00 | 코드 작성, 복잡한 추론, 일반 용도 |
| **Claude Opus 4.6** | ~$15.00 | ~$75.00 | 고난도 분석, 복잡한 에이전트 작업 |

> 💡 최신 가격은 [Anthropic 공식 가격 페이지](https://www.anthropic.com/pricing)에서 확인하세요.

**실제 비용 예시 — Haiku 기준:**
- 블로그 포스트 1건 요약 (~500 토큰 출력): 약 **$0.002**
- 하루 1,000건 고객 문의 분류: 약 **$0.80~$2.00**
- 코드 1,000줄 리뷰 (~3,000 토큰 출력): 약 **$0.012**

### 토큰 절약 전략

**1. 모델 라우팅:** 작업 복잡도에 따라 모델을 분리합니다.

```typescript
function selectModel(taskType: "classify" | "generate" | "analyze"): string {
  const modelMap = {
    classify: "claude-haiku-4-5-20251001",   // 저비용 분류
    generate: "claude-sonnet-4-6",           // 균형 잡힌 생성
    analyze: "claude-opus-4-6",              // 최고 품질 분석
  };
  return modelMap[taskType];
}
```

**2. 프롬프트 캐싱 활용 (Prompt Caching):** 동일한 시스템 프롬프트를 반복 사용하는 경우 캐싱을 활성화하면 **입력 비용을 90%까지 절감**할 수 있습니다.

```typescript
const response = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  system: [
    {
      type: "text",
      text: "매우 긴 시스템 프롬프트... (수천 토큰)",
      cache_control: { type: "ephemeral" }, // 캐시 지시어
    },
  ],
  messages: [{ role: "user", content: "질문" }],
});
```

**3. `max_tokens` 적절히 설정:** 필요 이상으로 크게 설정하면 낭비가 됩니다. 응답 길이 패턴을 분석해 적정값을 찾으세요.

## 6. 에러 처리 및 재시도

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({
  maxRetries: 3,      // 자동 재시도 (기본값: 2)
  timeout: 30_000,    // 타임아웃 30초 (기본값: 600초)
});

async function safeAsk(prompt: string): Promise<string | null> {
  try {
    const response = await client.messages.create({
      model: "claude-haiku-4-5-20251001",
      max_tokens: 512,
      messages: [{ role: "user", content: prompt }],
    });
    return (response.content[0] as { type: "text"; text: string }).text;
  } catch (err) {
    if (err instanceof Anthropic.APIError) {
      if (err.status === 429) {
        console.error("Rate limit 초과 — 잠시 후 재시도하세요.");
      } else if (err.status === 529) {
        console.error("API 과부하 상태 — 잠시 후 재시도하세요.");
      } else {
        console.error(`API 에러 ${err.status}:`, err.message);
      }
    }
    return null;
  }
}
```

**주요 에러 코드:**

| 상태 코드 | 의미 | 대응 방법 |
|----------|------|---------|
| 400 | 잘못된 요청 | 요청 파라미터 확인 |
| 401 | 인증 실패 | API 키 확인 |
| 429 | Rate limit 초과 | 지수 백오프 후 재시도 |
| 529 | 서버 과부하 | 잠시 대기 후 재시도 |

## 7. 자주 묻는 질문 (FAQ)

**Q1. Claude API와 ChatGPT API의 차이점은 무엇인가요?**  
Claude는 Anthropic의 Constitutional AI 방법론으로 훈련되어 안전성과 유용성을 동시에 추구합니다. 긴 컨텍스트 처리(최대 200K 토큰), 코딩 및 분석 능력에서 강점을 보입니다.

**Q2. API 키는 어디서 발급받나요?**  
[console.anthropic.com](https://console.anthropic.com)에서 회원가입 후 "API Keys" 메뉴에서 발급받을 수 있습니다.

**Q3. 무료로 사용할 수 있나요?**  
신규 가입 시 일정량의 무료 크레딧이 제공됩니다. 이후에는 사용량 기반으로 과금됩니다.

**Q4. 컨텍스트 윈도우는 얼마나 되나요?**  
Claude Sonnet 4.6과 Opus 4.6은 최대 **200,000 토큰**의 컨텍스트를 지원합니다. 약 15만 단어에 해당하는 분량입니다.

**Q5. 한국어 성능은 어떤가요?**  
Claude는 한국어를 포함한 다국어에서 높은 성능을 보입니다. 한국어 입력·출력 모두 영어와 유사한 수준으로 작동합니다.

**Q6. Rate limit(요청 한도)은 얼마나 되나요?**  
플랜에 따라 다르며, 일반적으로 분당 수천 건의 요청을 처리할 수 있습니다. 정확한 한도는 [Anthropic 사용 정책](https://www.anthropic.com/api)을 참조하세요.

**Q7. 스트리밍은 모든 모델에서 지원되나요?**  
네, Claude의 모든 모델에서 스트리밍 응답이 지원됩니다.

**Q8. API 응답 속도는 얼마나 빠른가요?**  
Haiku 모델의 경우 첫 토큰 응답 시간(TTFT)이 일반적으로 **200~500ms** 수준입니다. Sonnet은 약 500ms~1초, Opus는 1~2초 수준입니다.

**Q9. 프롬프트 캐싱이란 무엇인가요?**  
반복되는 긴 프롬프트(시스템 프롬프트, 문서 등)를 서버 측에서 캐싱하여 재전송 비용을 줄이는 기능입니다. 캐시 히트 시 입력 비용이 약 90% 절감됩니다.

**Q10. Python과 TypeScript 중 어떤 SDK를 추천하나요?**  
기능 면에서는 동일하지만, 백엔드/데이터 파이프라인이 목적이라면 Python, 웹 애플리케이션 통합이 목적이라면 TypeScript를 권장합니다.

**Q11. 대화 기록은 얼마나 유지되나요?**  
Claude API는 무상태(stateless)입니다. 대화 기록은 직접 배열로 관리해 매 요청에 포함시켜야 합니다.

**Q12. Anthropic API의 데이터 보안 정책은 어떻게 되나요?**  
기본적으로 API로 전송된 데이터는 모델 훈련에 사용되지 않습니다. 엔터프라이즈 플랜에서는 추가 데이터 보호 옵션이 제공됩니다.

## 마무리

Claude API로 AI를 통합하는 핵심 내용을 정리합니다:

- **시작**: `@anthropic-ai/sdk` 또는 `anthropic` 설치 → API 키 환경 변수 설정 → `messages.create()` 호출
- **스트리밍**: 사용자 경험을 위해 긴 응답에는 반드시 스트리밍 사용
- **비용**: 작업 복잡도에 맞게 Haiku / Sonnet / Opus를 선택해 모델 라우팅 구현
- **안전한 키 관리**: `.env` + 환경 변수, 절대 코드에 하드코딩 금지
- **에러 처리**: 429/529 에러에 대한 재시도 로직 필수

다음 단계로는 [Anthropic 공식 문서](https://docs.anthropic.com)에서 Batch API, Computer Use, Claude Code API 등 고급 기능을 살펴보시길 권장합니다.
